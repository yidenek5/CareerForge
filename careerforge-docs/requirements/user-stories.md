# CareerForge User Stories

## 1. Overview

User stories describe the main interactions between CareerForge users and the system.

They are written from the perspective of users to ensure that every feature provides real value and solves a specific problem.

The MVP contains three primary user roles:

- Guest User
- Job Seeker
- Employer

---

# 2. Guest User Stories

## Account Discovery

### US-GU-001

# 6. User Story Summary

| User Role | Main Goals |
|---|---|
| Guest User | Explore the platform and create an account |
| Job Seeker | Build profile, find jobs, apply, and track applications |
| Employer | Post jobs, review candidates, and manage hiring |
| Administrator | Maintain platform quality and security |

---

# 7. Definition of Success

A user story is successfully implemented when:

- The user can complete the intended action.
- The system provides the expected result.
- The feature solves the user's identified problem.
- The experience is simple and reliable.
